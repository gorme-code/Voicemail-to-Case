## Phase 2 Case field mapping (POST /api/voicemail)

| Salesforce Field | Value / Source |
|---|---|
| `Subject` | "Voicemail from {caller_name} — {caller_phone}" |
| `Origin` | `Phone` (hardcoded) |
| `Status` | `New` (hardcoded) |
| `OwnerId` | `00GWA000007oHmP2AU` (Queue ID, from env var) |
| **`RecordTypeId`** | **`012WA000001OMMzYAO`** (Voicemail record type — from env var) |
| `Callback_Number__c` | `caller_phone` from request payload |
| `Caller_Name__c` | `caller_name` from request payload |
| `External_Id__c` | Idempotency key generated by MuleSoft |
| `Description` | (blank — reserved for transcription) |

> **NEW vs. original requirements:** `RecordTypeId` was added in Phase 1 so the Voicemail
> layout can be scoped to voicemail Cases only. The Python API **must** set it on create.
> The integration/Connected App user's profile (or an assigned permission set) must have the
> Voicemail record type available, or the create call fails with INVALID_RECORD_TYPE.
